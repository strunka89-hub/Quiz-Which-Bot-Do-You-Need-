# Заявки из квиза → Telegram-бот

Это пошаговая инструкция, чтобы каждая заявка из квиза автоматически
приходила вам в личный Telegram в течение 1–2 секунд.

На всё уходит примерно **5 минут**. Программировать ничего не нужно —
весь код уже написан. Вам достаточно создать бота, узнать свой
`chat_id` и добавить два значения в настройки Vercel.

---

## Как это работает

```
Пользователь проходит квиз
        │
        ▼
    POST /api/lead          ← serverless-функция на Vercel
        │
        ▼
  Telegram Bot API (sendMessage)
        │
        ▼
   Ваш личный чат с ботом
```

Форма отправляется не напрямую в Telegram, а через нашу функцию
`api/lead.ts`. Это нужно, чтобы **токен бота оставался секретом**
и не попадал в браузер.

Пример сообщения, которое придёт:

```
🔥 Новая заявка из квиза

Имя: Иван
Телефон: +7 999 000-00-00
Сегмент: Теряются заявки

Ответы:
1. Как приходят клиенты: Везде и хаотично
2. Что раздражает: Теряю заявки
3. Как обрабатывает заявки: Вручную
4. Времени на общение: Очень много
5. Что хочет получить: Больше заявок

Источник: vk_community
Ссылка: https://vk.com/ushakova_ai
UTM: utm_source=vk, utm_campaign=cta_button
Время (МСК): 21.04.2026, 19:44
```

---

## Шаг 1. Создать Telegram-бота через @BotFather

1. Откройте Telegram и найдите в поиске контакт **@BotFather**
   (с синей галочкой).
2. Нажмите `Start` и отправьте команду `/newbot`.
3. BotFather спросит **имя бота** — это то, как он будет называться
   в чатах. Можно любое, например: `Квиз Ушаковой`.
4. Потом спросит **username** — должен быть уникальным и обязательно
   заканчиваться на `bot`. Например: `ushakova_quiz_bot`.
5. В ответ BotFather пришлёт сообщение с вашим токеном — строка вида:

   ```
   1234567890:AAH-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   ```

   **Это `TELEGRAM_BOT_TOKEN`. Сохраните его** — он нужен для Vercel.
   Никому не пересылайте: по нему можно писать от имени бота.

---

## Шаг 2. Получить свой `chat_id`

Боту нельзя просто так написать вам первым — сначала **вы** должны
открыть с ним диалог.

1. В Telegram откройте вашего только что созданного бота
   (в сообщении от BotFather есть ссылка `t.me/ushakova_quiz_bot`).
2. Нажмите `Start` (или отправьте любое сообщение, например, `привет`).
3. Теперь откройте в обычном браузере (не в Telegram!) ссылку,
   подставив ваш токен:

   ```
   https://api.telegram.org/bot<ВАШ_ТОКЕН>/getUpdates
   ```

   Пример:

   ```
   https://api.telegram.org/bot1234567890:AAH-xxx.../getUpdates
   ```

4. В ответе (это будет JSON-текст) найдите блок `"chat"`:

   ```json
   "chat": {
     "id": 123456789,
     "first_name": "Анна",
     "type": "private"
   }
   ```

5. Число `id` (здесь — `123456789`) — это ваш **`TELEGRAM_CHAT_ID`**.
   Сохраните его.

> Если в ответе пусто (`"result":[]`) — значит, вы ещё не написали
> боту. Напишите ему любое сообщение и обновите страницу.

---

## Шаг 3. Добавить переменные окружения в Vercel

1. Откройте [vercel.com](https://vercel.com/) и войдите в свой проект
   с квизом.
2. Перейдите: **Settings → Environment Variables**.
3. Добавьте **три переменные**. Для каждой отметьте все три окружения:
   `Production`, `Preview`, `Development`.

   | Key | Value |
   | --- | --- |
   | `TELEGRAM_BOT_TOKEN` | токен из шага 1 |
   | `TELEGRAM_CHAT_ID` | число из шага 2 |
   | `VITE_WEBHOOK_URL` | `/api/lead` |

   > `VITE_WEBHOOK_URL` можно не добавлять — по умолчанию используется
   > `/api/lead`. Но лучше задать явно, чтобы на Preview-деплое точно
   > работало так же.

4. После добавления переменных нужен **redeploy**, чтобы Vercel
   подхватил новые значения:

   - Перейдите во вкладку **Deployments**.
   - У последнего production-деплоя нажмите на три точки `⋯`.
   - Выберите **Redeploy**.

---

## Шаг 4. Проверить, что всё работает

1. Откройте задеплоенный URL квиза с телефона или в режиме
   инкогнито.
2. Пройдите квиз до конца и отправьте тестовую заявку (имя — любое,
   телефон — свой или `+7 999 000-00-00`).
3. В течение нескольких секунд в ваш Telegram должно прийти
   сообщение от бота.

Если сообщение не пришло — см. раздел «Если что-то не работает».

---

## Если что-то не работает

### Пользователь видит ошибку «Не удалось отправить»

Откройте на Vercel: **ваш проект → Logs → Functions → /api/lead**.
Посмотрите последнюю запись.

- `missing TELEGRAM_BOT_TOKEN or TELEGRAM_CHAT_ID` — переменные
  не добавлены или сделан не тот redeploy. Вернитесь к шагу 3.
- `telegram error 401` — неверный токен, пересоздайте бота или
  скопируйте токен заново.
- `telegram error 400 chat not found` — неверный `chat_id` или вы
  ещё не написали боту (шаг 2, пункт 2).
- `telegram error 403 bot was blocked by the user` — вы заблокировали
  бота в Telegram. Разблокируйте и снова нажмите `Start`.

### Заявка отправляется, но ничего не приходит

Проверьте, что в браузере запрос уходит на правильный URL:

- Откройте DevTools → Network → найдите запрос `lead`.
- Status должен быть `200`, в Response — `{"ok":true}`.
- Если Status `404` — значит, в Vercel не задеплоилась папка `api/`.
  Сделайте `git push` и дождитесь деплоя.

### Хотите получать заявки в группу / канал, а не в личку

1. Создайте Telegram-группу, добавьте туда вашего бота.
2. В настройках группы дайте боту право писать сообщения.
3. В поиске BotFather'а включите у бота Privacy Mode OFF:
   `/mybots → выбрать бота → Bot Settings → Group Privacy → Turn off`.
4. Напишите в группу что-нибудь с упоминанием бота (`@ushakova_quiz_bot`).
5. Снова откройте `https://api.telegram.org/bot<ТОКЕН>/getUpdates` —
   найдите `"chat":{"id": -1001234567890, ...}`. Число со знаком
   минус — это `chat_id` группы. Подставьте его в Vercel вместо
   личного `TELEGRAM_CHAT_ID`.

### Хотите двух получателей (например, себя и ассистента)

Проще всего — создать группу из двух человек и бота, и использовать
`chat_id` этой группы (см. пункт выше).

## Переменные окружения — сводная таблица

| Переменная | Где задаётся | Секретная | Для чего |
| --- | --- | --- | --- |
| `VITE_WEBHOOK_URL` | Vercel (Public) | нет | URL, куда фронт шлёт лид. По умолчанию `/api/lead`. |
| `TELEGRAM_BOT_TOKEN` | Vercel | **да** | Токен бота от @BotFather. |
| `TELEGRAM_CHAT_ID` | Vercel | **да** | ID чата, куда бот шлёт заявки. |

`TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_ID` **не** начинаются с `VITE_`
— это важно. Без префикса `VITE_` Vite не включает переменную в
браузерный бандл, и она остаётся доступной только серверной
функции `api/lead.ts`.

## Что дальше

- Получаете заявки в Telegram → отвечаете клиенту вручную или
  пересылаете в CRM.
- Если в будущем захотите подключить CRM (AmoCRM, Bitrix, Tilda,
  Google Sheets) — достаточно поменять `VITE_WEBHOOK_URL` на URL
  своего webhook'а. Telegram при этом можно оставить параллельно,
  просто добавив второй `fetch` в `api/lead.ts`.
